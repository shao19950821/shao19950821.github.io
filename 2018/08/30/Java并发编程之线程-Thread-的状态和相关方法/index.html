Java中线程的六大状态

在Java中，线程的状态是Thread类里的一个枚举类型来描述的。这个枚举类型一共六个值，分别是:NEW、RUNNABLE、BLOCKED、TIMED_WAITING、WAITING、TERMINATED。
下面就是Thread类描述线程状态的源码。

根据这个枚举里提供的Javadoc文档注释，我们大致了解一下这些状态的具体描述：

1.NEW(初始):新创建了一个线程对象，但还没有调用start()方法。

RUNNABLE(运行):很多文章都有提到过线程在运行前有个READY(就绪),运行中则是RUNNING(运行中)，Java中将READY和RUNNING笼统的称为RUNNABLE。创建了线程对象后，其他线程(比如主线程main)调用了该线程对象的start()方法。但是此时线程只是处于可运行状态，需要得到操作系统的其他资源，如被调度器选中，获取CPU的使用权。在得到了CPU的时间片后，线程才变成了RUNNING(运行中)状态。

3.BLOCKED(阻塞):表示线程阻塞于monitor lock也就是内置锁，线程需要拿到monitor lock才能进入同步方法或者同步代码块。

4.WAITING(等待状态):线程因为调用如下方法进入了等待状态：Object类的wait()方法，Thread类的join()方法，LockSupport的park()方法，在这个状态的线程需要等待其他线程做出特殊的动作(比如通知或者中断)。

5.超时等待(TIMED_WAITING)：该状态不同于WAITING，它可以在指定的时间后自行返回。通常调用了Object类的带参数的wait(long)方法，或者是Thread类带参数的join(long)方法，还有就是LockSupport.parkNanos()和 LockSupport.parkUntil()方法，这两个我们用的比较少。

6.终止(TERMINATED)：表示该线程已经执行完毕。

线程相关方法的详细介绍

对于线程的大致方法，我之前一直很模糊，特别是yield()、join()、notify()、notifyAll()这些方法，所以，这里针对每个方法做一下介绍。

sleep()和wait()方法

这两个方法大致上都了解，大致说一下他们的不同：
1.sleep()是Thread类的方法，wait()是Object类的方法；
2.sleep()不会释放对象锁，即使当前线程使用sleep()方法让出了cpu，但其他被同步锁挡住了的线程也无法得到执行，而wait()会释放对象锁，所以wait()只能在同步控制方法或者同步控制块里面使用，而sleep可以在任何地方使用；
3.sleep()方法抛出了InterruptedException异常，所以使用它必须捕获异常，但是wait()方法就不必捕获异常；
4.sleep方法必须传入参数，用于表示睡眠的时间，单位是毫秒，但是wait()方法可以不必传入参数，调用wait()方法的线程必须要通过notify()和notifyAll()方法唤醒。

但是有一点必须注意的是，sleep()方法是Thread类的静态方法，所以即使是在a线程里调用了b线程的sleep方法，最终sleep的仍然是a。。

interrupt、interrupted、isInterrupted方法

把这三个方法放一起讲，是因为它们都涉及到线程的中断。
interrupt()方法只是设置一个中断标记，如果对处于阻塞状态(调用了sleep、wait、join方法)的线程调用interrupt方法时，会抛出InterruptException异常，而这个异常会清除中断标记。

yield方法

yield()方法也是Thread类的静态方法，它的作用是暂停当前正在执行的线程对象(及放弃当前拥有的CPU资源)，并执行其他的线程。
yield的作用是让当前线程回到可运行的状态，从而可以使具有相同优先级的线程获得运行的机会，目的是让相同优先级的线程之间能适当的轮转执行。但是，调用yield()方法不一定就能达到让步的目标，因为让步的线程还有可能被重新选中。

join方法

join()方法，就是把指定的线程加入到当前的线程，将两个交替执行的线程合并为顺序执行的线程。

几个方法的总结

1.Thread.sleep(long millis)，是Thread类的静态方法，所以一定是当前线程调用此方法，当前线程进入TIMED_WAITING状态，但不释放对象锁，millis后线程自动苏醒进入就绪状态。作用：给其它线程执行机会的最佳方式。

2.Thread.yield()，是Thread类的静态方法，一定是当前线程调用此方法，当前线程放弃获取的CPU时间片，但不释放锁资源，由运行状态变为就绪状态，让OS再次选择线程。作用：让相同优先级的线程轮流执行，但并不保证一定会轮流执行。实际中无法保证yield()达到让步目的，因为让步的线程还有可能被线程调度程序再次选中。Thread.yield()不会导致阻塞。该方法与sleep()类似，只是不能由用户指定暂停多长时间。

3.t.join()/t.join(long millis)，当前线程里调用其它线程t的join方法，当前线程进入WAITING/TIMED_WAITING状态，当前线程不会释放已经持有的对象锁。线程t执行完毕或者millis时间到，当前线程进入就绪状态。

4.obj.wait()，当前线程调用对象的wait()方法，当前线程释放对象锁，进入等待队列。依靠notify()/notifyAll()唤醒或者wait(long timeout) timeout时间到自动唤醒。

5.obj.notify()唤醒在此对象监视器上等待的单个线程，选择是任意性的。notifyAll()唤醒在此对象监视器上等待的所有线程。
